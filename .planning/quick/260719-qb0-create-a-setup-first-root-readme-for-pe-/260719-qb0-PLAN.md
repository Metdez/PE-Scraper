# Quick Task 260719-qb0: Setup-first project README

## Goal

Create a polished root README that gets a Windows user from clone to a verified first run, documents the complete supported CLI surface, and explains optional discovery and NanoClaw automation without making either a prerequisite.

## Task 1: Write the codebase-grounded README

**Files**

- Create: `README.md`
- Read for evidence: `pyproject.toml`, `src/pescraper/cli.py`, `src/pescraper/doctor.py`, `src/pescraper/db.py`, `src/pescraper/exporter.py`, `integrations/searxng/docker-compose.yml`, `integrations/searxng/settings.yml`, `integrations/nanoclaw/pe-scraper/CLAUDE.md`, `integrations/nanoclaw/pe-scraper/run-pescraper.sh`, `integrations/nanoclaw/pe-scraper/run-pescraper.mjs`, `scripts/nanoclaw_bridge.py`, `scripts/start-nanoclaw-bridge.ps1`, and `scripts/install-heartbeat.ps1`

**Action**

Write a setup-first README with:

1. A concise value proposition and honest capability summary.
2. Required Windows prerequisites: Git, Python 3.11 or 3.12, `uv`, Ollama, `qwen3:4b`, and Crawl4AI's browser setup.
3. A copy-paste quick start using `uv sync`, `uv run crawl4ai-setup`, `ollama pull qwen3:4b`, `uv run pescraper doctor`, and a first `run-firm` command.
4. CLI recipes for CSV batches, status, stored-record lookup, structured queries, export, benchmark, and heartbeat.
5. An explicit explanation of `PESCRAPER_DB`, the SQLite source of truth, default export paths, confidence, provenance, and `needs_review` behavior.
6. Optional SearXNG startup and discovery instructions based on the checked-in compose file.
7. Optional NanoClaw integration instructions that accurately describe creation and placement of the bridge token, the project mount, bridge startup and `/healthz` check, the container wrapper route, and the allowed command boundary, without inventing missing upstream NanoClaw installation steps.
8. Troubleshooting keyed to `pescraper doctor`, Ollama, Crawl4AI/Chromium, SearXNG JSON, database location, and failed queue jobs.
9. Development/test commands and a short “Using an AI coding assistant” section with a safe ready-made prompt that tells the assistant to inspect the repository and preserve user data.

Keep the document scannable, copy-paste friendly, and free of GSD methodology or unverified marketing claims.

**Verify**

- Confirm every `pescraper` command and option in the README against fresh `--help` output.
- Confirm all paths linked from the README exist.
- Confirm setup versions and dependencies against `pyproject.toml`.
- Confirm optional integration commands against their checked-in scripts and compose configuration.

**Done**

- `README.md` exists at the repository root.
- Required and optional setup are unmistakably separated.
- A user has a linear path from clone through health check to first scrape and export.
- All major CLI workflows and operational failure paths are covered.
