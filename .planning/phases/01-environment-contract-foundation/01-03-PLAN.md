<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| doctor -> localhost:11434 (Ollama) | The smoke test calls the local model service; only localhost is contacted |
| Crawl4AI/Chromium -> inline raw HTML | The Chromium launch renders inline HTML (raw scheme), so no untrusted external site is fetched during the smoke test |
| doctor -> subprocess (crawl4ai-doctor) | A child process is spawned to verify the browser toolchain |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-06 | Denial of Service | check functions hanging on a down service (unattended re-run wedges) | medium | mitigate | Each check wraps failures as ok=False and never raises; the Ollama client uses default timeouts and the Chromium launch uses inline raw HTML with no network wait |
| T-01-07 | Tampering | Scraped/model content echoed to a Windows console (mojibake / injection) | low | mitigate | UTF-8 stdout is forced at package import; the smoke test uses a tiny fixed prompt and inline HTML — no untrusted web content is rendered here |
| T-01-08 | Elevation of privilege | crawl4ai-doctor subprocess execution | low | accept | Invoked from the project's own venv with no user-supplied arguments; local developer context only |
</threat_model>
