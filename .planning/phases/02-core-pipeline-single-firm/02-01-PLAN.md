<objective>
Implement the three pure-function "business rules" modules that decide how extracted/seeded
data is merged, scored, and traced back to its source page — with zero I/O, zero LLM calls,
zero crawl4ai/ollama imports. These are the deterministic decision points CONTEXT.md and
ROADMAP require to never be left to the model: null-safe merging (DATA-04), code-computed
confidence (PIPE-04), and code-side quote-to-page matching (PIPE-05's provenance mechanism).

Purpose: Every later plan (ingest, integration) needs these three modules as a stable,
independently-testable contract. Building them first, in isolation, with no dependency on
crawl/extract, lets Wave 1 run fully in parallel.
Output: `src/pescraper/merge.py`, `src/pescraper/confidence.py`, `src/pescraper/provenance.py`,
each with a corresponding fast offline pytest module.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Extracted/seed values -> merge.merge_firm_record | Values ultimately originate from scraped web content or an untrusted CSV; this module treats them as opaque data, never executes or interprets them |
| Quote text -> provenance.find_source_page | Quote strings originate from LLM output (which itself echoes scraped page content); matching is read-only string comparison |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Tampering | merge.merge_firm_record | low | accept | Pure dict/attribute manipulation on already-validated Pydantic models; no eval/exec/SQL — a maliciously crafted value can at worst become a wrong-but-typed field value, caught later by confidence/needs_review scoring |
| T-02-02 | Denial of Service | provenance.find_source_page | low | mitigate | `difflib.SequenceMatcher.quick_ratio()` is O(n) amortized (not the slower `ratio()`), and quotes/pages are bounded by extract.py's char budgets (02-04) upstream — no unbounded input reaches this function in the real pipeline |
</threat_model>
